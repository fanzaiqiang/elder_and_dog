## 一、執行摘要

### 1.1 提案概述

建議導入 **NVIDIA Jetson Orin Nano SUPER Developer Kit (8GB)** 作為機器狗的「本地大腦」，結合 **OpenClaw** 個人 AI 助理框架，實現：

- **多渠道互動介面**（WhatsApp/Telegram/Slack/Web/TUI）
- **邊緣即時控制**（避障導航 <200ms）
- **雲地協作 AI**（簡單任務本地、複雜推理上雲）
- **個人化記憶系統**（使用者偏好自動記錄）

### 1.2 核心價值

| 指標 | 當前架構 (v4.0) | 新架構 (v5.0) |
|------|----------------|---------------|
| **使用者介面** | 需自建 Web UI | ✅ 免費獲 6+ 種通訊軟體整合 |
| **移動部署** | 僅限實驗室 | ✅ 可攜式邊緣運算 |
| **記憶系統** | 需自建資料庫 | ✅ OpenClaw 內建記憶 |
| **語音互動** | 需自建 STT/TTS | ✅ 內建 Voice Wake + TTS |
| **硬體成本** | Mac VM 持續成本 | ✅ 一次性 $500 |

### 1.3 建議決議

**建議：採用此方案**，但必須完成以下先決條件：
1. 自建 Sensor Gateway 中介層（關鍵技術組件）
2. 部署 Tailscale 安全網路隔離
3. 建立資源監控與優雅降級機制

### 3.2 關鍵技術決策

#### 決策 1：為何需要 Sensor Gateway？

**問題**：為何不直接讓 OpenClaw 訂閱 ROS2 topics？

**原因**：
- 原始 LiDAR 點雲：~10MB/s → 壓垮 8GB 記憶體
- 原始深度影像：~5MB/s → 網路延遲 >200ms
- Agent 不需要原始資料，只需要「決策級資訊」

**解決方案**：
```
ROS2 原始感測器 (10MB/s)
    ↓
Perception Bridge (降採樣/聚類/特徵提取)
    ↓
Sensor Gateway (ring buffer + API)
    ↓
OpenClaw 接收 (JSON: 障礙物列表、距離統計，~1KB)
```

**資料轉換範例**：
| 原始資料 | 處理後 | 大小 |
|---------|--------|------|
| LiDAR 點雲 (10MB/s) | 障礙物列表 (JSON) | ~1KB |
| 深度影像 (5MB/s) | 距離統計 (JSON) | ~500B |
| RGB 影像 (10MB/s) | JPEG keyframe | ~50KB |

#### 決策 2：快系統 <200ms 如何實現？

**關鍵原則**：快反應**不能**經過 OpenClaw/LLM！

**分層架構**：
| 層級 | 延遲 | 責任 | 實現位置 |
|------|------|------|----------|
| **Layer 0** | <50ms | 緊急停止/碰撞避免 | ROS2 Safety Node (本地) |
| **Layer 1** | <200ms | 避障導航/路徑規劃 | Nav2 (本地) |
| **Layer 2** | 500ms-2s | 任務策略/目標選擇 | OpenClaw Agent |
| **Layer 3** | 2-5s | 複雜語意/VLM | GPU Server |

**運作流程**：
1. 使用者：「去客廳找沙發」
2. OpenClaw 解析意圖 → 發送高階目標給 Nav2
3. Nav2 本地規劃路徑並執行（<200ms 反應）
4. Safety Node 持續監測 LiDAR，遇障礙立即反應（<50ms）
5. OpenClaw 監控進度，必要時調整策略（不需要快反應）

## 四、技術實施細節

### 4.1 OpenClaw 與 ROS2 整合

**控制鏈路**：
```
使用者訊息 (Telegram)
    ↓
OpenClaw Gateway
    ↓ system.run tool
ros-mcp-server
    ↓ MCP Protocol
rosbridge_server
    ↓ WebSocket
ROS2 Topics/Services
    ↓
go2_robot_sdk
    ↓
Go2 機器狗執行動作
```

**程式碼範例**：
```json
// OpenClaw 執行 ROS2 指令範例
{
  "tool": "system.run",
  "params": {
    "command": "ros2 topic pub /cmd_vel geometry_msgs/Twist '{linear: {x: 0.1}, angular: {z: 0.0}}' --once",
    "timeout": 5000
  }
}

// OpenClaw 發送動作指令
{
  "tool": "system.run", 
  "params": {
    "command": "ros2 topic pub /webrtc_req go2_interfaces/msg/WebRtcReq '{api_id: 1016, topic: \"rt/api/sport/request\"}' --once",
    "timeout": 3000
  }
}
```

### 4.2 資源配置評估

**Jetson Orin Nano SUPER 8GB 分配**：

| 組件 | 預估記憶體 | 備註 |
|------|-----------|------|
| Jetson OS + 系統 | ~1.5GB | 作業系統 overhead |
| OpenClaw Gateway | ~500MB-1GB | Node.js runtime |
| ROS2 Humble core | ~500MB | 基本節點 |
| Nav2 + SLAM | ~1-1.5GB | 導航堆疊 |
| RealSense ROS2 | ~300-500MB | 深度 + RGB |
| coco_detector | ~300-500MB | MobileNet 模型 |
| Sensor Gateway | ~200-300MB | 中介層服務 |
| **預留緩衝** | ~2GB | GPU 共享記憶體 |
| **總計** | **~6.5-7.5GB** | ✅ **在 8GB 範圍內** |

**結論**：8GB **夠用**，但需要優化配置。

### 4.3 快/慢系統分工

**快系統 (Jetson 8GB) - 必須本地**：
| 功能 | 技術 | 延遲 |
|------|------|------|
| 避障導航 | Nav2 + SLAM | <200ms |
| 緊急停止 | Safety Layer | <50ms |
| 物件偵測 | coco_detector | <150ms |
| 深度估計 | RealSense D435 | <30ms |
| 簡單指令理解 | 小型 LLM (3-4B) | <500ms |

**慢系統 (GPU Server RTX 6000) - 遠端呼叫**：
| 功能 | 技術 | 延遲 | 觸發條件 |
|------|------|------|----------|
| 複雜語意理解 | Qwen2.5-72B | 2-5s | 模糊/多義指令 |
| 視覺場景理解 | LLaVA-34B | 3-5s | 需要詳細描述 |
| 精準語音轉文字 | Whisper Large V3 | 1-2s | 音訊輸入時 |
| 精確物件追蹤 | SAM2 | ~22ms | 需要追蹤時 |

**自動分流邏輯**：
```python
# OpenClaw Agent 決策邏輯
if task_complexity == "high" or needs_visual_understanding:
    # 呼叫 GPU Server
    result = http_post("http://gpu-server:8000/vlm", {
        "image": keyframe_jpeg,
        "prompt": "描述場景並識別物品位置"
    })
else:
    # 本地處理
    result = local_llm_qwen_3b(process(prompt))
```

## 七、風險評估與緩解

### 7.1 高風險項目

#### 風險 1：OpenClaw 安全漏洞 ⚠️⚠️⚠️

**風險描述**：
- OpenClaw 近期爆發 1,800+ 暴露實例
- 洩漏 API 金鑰、聊天歷史、帳號憑證
- 預設配置可能允許未授權存取

**影響**：高（資料外洩、未授權控制機器狗）

**緩解措施**：
1. ✅ **網路隔離**：僅允許區網存取（192.168.x.x）
2. ✅ **Tailscale**：使用 tailnet-only 模式，不暴露公開 IP
3. ✅ **密碼驗證**：強制設置 `gateway.auth.mode: "password"`
4. ✅ **DM 配對**：啟用 `dmPolicy="pairing"`，只允許白名單使用者
5. ✅ **Firewall**：關閉所有非必要 ports

#### 風險 2：記憶體壓力導致 OOM ⚠️⚠️

**風險描述**：
- 8GB 記憶體同時運行多個服務
- 可能發生 OOM（Out of Memory）導致系統崩潰

**影響**：中（系統不穩定）

**緩解措施**：
1. ✅ **Docker 限制**：為每個容器設置記憶體上限
2. ✅ **zram**：啟用壓縮交換空間
3. ✅ **監控告警**：Prometheus + Grafana 監控記憶體使用
4. ✅ **優雅降級**：記憶體不足時自動關閉非關鍵功能（如視覺化）

### 7.2 中風險項目

#### 風險 3：延遲超標 ⚠️

**風險描述**：
- 感測器資料處理鏈路延遲可能超過 200ms
- 影響避障反應時間

**緩解措施**：
1. ✅ **分層處理**：快反應留在本地（Layer 0/1），不經過 OpenClaw
2. ✅ **資料降採樣**：只傳精簡表徵，不傳原始點雲
3. ✅ **快取策略**：常用資料預先載入記憶體

#### 風險 4：OpenClaw 專案穩定性 ⚠️

**風險描述**：
- OpenClaw 是很新的專案（2 個月前發布）
- API 可能變動，文件可能不完整

**緩解措施**：
1. ✅ **版本鎖定**：使用特定版本（不追最新 dev 版）
2. ✅ **原始碼研究**：預留時間研究原始碼理解內部機制
3. ✅ **備案方案**：若 OpenClaw 不穩定，可退回 ROS2 + 自建 Web UI

### 7.3 風險總表

| 風險 | 等級 | 機率 | 影響 | 緩解後風險 |
