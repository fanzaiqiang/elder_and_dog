# GEMINI.md - Context & Instructions for Gemini Agents

## 1. Project Overview
**Project Name:** Elder and Dog (老人與狗) / Go2 Smart Item Finding System
**Goal:** Develop an autonomous item-finding assistant for the elderly using the Unitree Go2 robot.
**Core Features:**
- **Autonomous Navigation:** SLAM (`slam_toolbox`) + Navigation (`nav2`).
- **Perception:** Real-time object detection (`coco_detector`) using TorchVision (MobileNet/FasterRCNN).
- **Interaction:** Voice interaction (TTS/STT) and Web UI (Foxglove).
- **Platform:** Ubuntu 22.04 (ROS 2 Humble) on Unitree Go2 (or Isaac Sim).

## 2. Development Environment & Architecture
**Topology:**
- **Compute Node (Mac/VM):** Runs ROS 2 core, driver, SLAM, Nav2.
- **Control Station (Windows):** Runs VS Code (SSH), Foxglove Studio (Visualization).
- **Robot (Go2):** Connects via WebRTC (Wi-Fi) or CycloneDDS (Ethernet).

**Directory Structure:**
- `go2_robot_sdk/`: Main Python driver (Clean Architecture).
- `go2_interfaces/`: Custom ROS 2 `.msg` definitions.
- `lidar_processor/`: LiDAR point cloud processing (Python).
- `lidar_processor_cpp/`: High-performance LiDAR processing (C++).
- `coco_detector/`: Object detection node (PyTorch).
- `scripts/`: Utility scripts (e.g., `phase1_test.sh`).

## 3. Key Commands

### Build
```bash
# Build all packages
colcon build

# Build specific package
colcon build --packages-select go2_robot_sdk
```

### Run
**Option 1: Automated Testing (Preferred)**
```zsh
# Phase 1 Test Suite (Drive, Scan, SLAM, Nav2)
zsh phase1_test.sh env       # Check environment
zsh phase1_test.sh t1        # Start Driver
zsh phase1_test.sh t2        # Monitor Frequencies
zsh phase1_test.sh t3        # Start SLAM + Nav2
zsh phase1_test.sh t4        # Interactive Control
```

**Option 2: Manual Launch**
```bash
source install/setup.zsh
export ROBOT_IP="192.168.12.1"
export CONN_TYPE="webrtc"

# Full Launch
ros2 launch go2_robot_sdk robot.launch.py slam:=true nav2:=true foxglove:=true

# Minimal Driver Only
zsh start_go2_simple.sh
```

### Dependencies
- **Python:** Use `uv pip install -r src/requirements.txt` (Strict Rule: Always use `uv pip install`).
- **ROS:** `rosdep install --from-paths src --ignore-src -r -y`.
