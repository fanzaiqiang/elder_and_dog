- 專案目前以 WASM (`libvoxel.wasm`) 解碼 LiDAR

## ✅ 解法與處理

### A) 固定有線網段到 192.168.123.x
- 修改 `enp0s2` 為 `192.168.123.100/24`
- 透過 `netplan` 固定設定，避免每次開機手動改

**指令：**
```bash
sudo ip addr del 192.168.1.200/24 dev enp0s2
sudo ip addr add 192.168.123.100/24 dev enp0s2
ping -c 4 192.168.123.161
```

**永久設定（netplan）：**
```yaml
network:
  version: 2
  ethernets:
    enp0s1:
      dhcp4: true
    enp0s2:
      dhcp4: no
      addresses:
        - 192.168.123.100/24
```

```bash
sudo netplan apply
```

> `Cannot call Open vSwitch: ovsdb-server.service is not running.` 可忽略。

### B) LZ4 LiDAR 解碼器最小移植
- 新增 LZ4 解碼器檔案
- `data_decoder.py` 優先使用 LZ4，失敗則 fallback 回 WASM
- `requirements.txt` 加入 `lz4>=4.0.0`

**新增/修改：**
- `go2_robot_sdk/infrastructure/sensors/lidar_decoder_lz4.py`
- `go2_robot_sdk/infrastructure/webrtc/data_decoder.py`
- `requirements.txt`

**確認：**
- `disableTrafficSaving(True)` 已在 WebRTC 連線後被呼叫（`webrtc_adapter.py:55`）

**安裝依賴：**
```bash
uv venv .venv
source .venv/bin/activate
uv pip install lz4
```
