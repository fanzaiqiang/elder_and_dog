## 📋 診斷清單（按優先級排序）

### Phase 0: 基礎驗證（30 分鐘）

```bash
# 環境設定
export ROBOT_IP="192.168.123.161"
export CONN_TYPE="webrtc"
source /opt/ros/humble/setup.bash && source install/setup.bash

# 1. 感測器頻率檢查
ros2 topic hz /point_cloud2   # 目標: ≥10 Hz
ros2 topic hz /scan           # 目標: ≥5 Hz
ros2 topic hz /odom           # 目標: ≥20 Hz
ros2 topic hz /imu            # 目標: ≥50 Hz

# 2. TF 樹完整性
ros2 run tf2_tools view_frames  # 產生 frames.pdf
ros2 run tf2_ros tf2_echo map base_link  # 應顯示有效轉換

# 3. SLAM 節點狀態
ros2 node list | grep slam
ros2 topic info /scan  # 確認有 subscriber (slam_toolbox)
```

### Phase 1: 資料品質（1 小時）

```bash
# 4. /scan 資料有效性
ros2 topic echo /scan --once
# 檢查: ranges 應有實際距離值，非全 inf

# 5. /odom 漂移測試
# 機器人原地不動 30 秒，記錄起始和結束 pose
ros2 topic echo /odom --field pose.pose.position --once  # 起始
# 等待 30 秒
ros2 topic echo /odom --field pose.pose.position --once  # 結束
# 計算差值，應 <5cm

# 6. pointcloud_to_laserscan 轉換比對
ros2 topic echo /point_cloud2 --field width --once  # 點數
ros2 topic echo /scan --field ranges --once | wc -w  # 有效點數
# 比例應 >50%
```

### Phase 2: SLAM 運作（1-2 小時）

```bash
# 7. SLAM feedback
ros2 topic echo /slam_toolbox/feedback --once
# 應有輸出，否則 SLAM 沒在運作

# 8. /map 狀態
ros2 topic echo /map --field info --once
ros2 topic echo /map --field data --once | head -100
# 應有 0 (空閒) 和 100 (佔據)，非全 -1

# 9. 小空間測試
# 在特徵豐富的小房間（有傢俱、牆角）測試
# 讓機器狗慢速移動，觀察地圖建立

# 10. 迴圈閉合測試
# 繞房間一圈回到原點，觀察地圖是否修正
```

## Part 2: Go2 Pro 感測器能力盤點

### 內建感測器

| 感測器 | 規格 | 用途 | 限制 |
|--------|------|------|------|
| **4D LiDAR L1** | 360°×90° FOV, 21,600 samples/s, min 0.05m | SLAM、避障 | 低成本有噪聲，**無法偵測 <0.3m 障礙** |
| **前置相機** | 1080P@15fps, F2.2, 120° FOV | 影像辨識 | 無深度資訊 |
| **IMU** | 集成在 LiDAR 內 | 姿態估計 | 需校準 |
| **深度相機** | 僅標註有，規格不明 | 3D 環境 | Go2 Pro 實測無法使用 |

### L1 LiDAR 詳細規格

- **採樣率：** 21,600 次/秒
- **視野：** 360° × 90° 半球形
- **最小偵測距離：** 0.05m（極小盲區）
- **4D 含義：** 3D 位置 + 1D 灰度（強度）

### 已知限制（CMU autonomy_stack_go2 確認）

> "L1 lidar 是低成本感測器，噪聲相對較高。該感測器無法區分低障礙物"

⚠️ **重要：** 所有障礙物需 **>0.3m 高度** 才能被偵測

## Part 4: Jetson Orin Nano + RealSense D435 整合方案

### 硬體選擇建議

#### Jetson Orin Nano

| 型號 | VRAM | 價格 | 適用場景 |
|------|------|------|----------|
| 8GB | 8GB | ~$14,800 | 輕量 AI + 遠端 GPU 協作 |
| 16GB | 16GB | ~$28,000+ | 全本地運算 |

**會議決議：** 先測試 8GB 模型能力，再決定

#### RealSense D435 vs D435i

| 項目 | D435 | D435i |
|------|------|-------|
| 深度 | ✅ | ✅ |
| RGB | ✅ | ✅ |
| IMU | ❌ | ✅ |
| SLAM 適用性 | 需外部 IMU | 建議，內建 IMU |
| FOV | 87° × 58° | 87° × 58° |
| 有效距離 | 0.3-3m | 0.3-3m |

**目前狀態：** 你們有 D435（無 IMU）

### D435 無 IMU 的因應方案

1. **使用 Go2 內建 IMU**
   - Go2 的 L1 LiDAR 內建 IMU
   - 可透過 `/imu` topic 取得資料
   - 需處理時間戳同步問題

2. **使用 LiDAR SLAM 而非 VSLAM**
   - nvblox 可以用 LiDAR 資料建圖
   - 不依賴相機 IMU
   - D435 只做深度感知 + 物件偵測

3. **考慮升級到 D435i（建議）**
   - 價差約 $30-50 USD
   - 內建 IMU 可同步深度資料，VSLAM 效果更好
   - 若預算允許，建議採購 D435i

### NVIDIA Isaac ROS 方案

官方支援的 SLAM 方案：

| 方案 | 說明 | 優勢 |
|------|------|------|
| **cuVSLAM** | GPU 加速視覺 SLAM | 30+ FPS，支援多相機 |
| **nvblox** | 3D 重建 + costmap | Vision-only 避障 |
| **Isaac Perceptor** | 完整感知堆疊 | Lidar-free 導航 |

**cuVSLAM + RealSense 整合：**
- 官方教學：https://nvidia-ai-iot.github.io/jetson_isaac_ros_visual_slam_tutorial/
- 預配置 SD 卡映像，最快速評估

**nvblox 避障：**
- 處理深度 + pose → 3D 場景重建
- 輸出 2D costmap 給 Nav2
- Vision-only 3D 避障，不需 LiDAR

### 已知問題與解決方案

| 問題 | 說明 | 解決方案 |
|------|------|----------|
| RealSense 連接不穩 | 部分用戶報告 D435i 在 Orin Nano 上連接問題 | 使用 JetsonHacks 教學，user-space SDK |
| 效能限制 | 某些專案需限制 15Hz 避免崩潰 | 降低解析度或頻率 |
| RTABMAP 資源消耗 | SLAM 運行時 Jetson 可能過載 | 優化參數或用 cuVSLAM |

## Part 5: 建議的系統架構

根據調查結果，建議架構：

```
┌─────────────────────────────────────────────────────────┐
│                    GPU Server (雲端)                     │
│         VLA/VLM (決策)、大型 LLM (複雜推理)              │
│         Qwen2.5-72B / LLaVA-34B / Whisper V3            │
└──────────────────────────┬──────────────────────────────┘
                           │ HTTP API (慢系統 1-5s)
                           ↓
┌─────────────────────────────────────────────────────────┐
│         Jetson Orin Nano 8GB (背在狗上)                  │
│  ┌─────────────────────────────────────────────────┐    │
│  │ RealSense D435i (深度 + RGB + IMU)              │    │
│  │ → cuVSLAM (視覺 SLAM) 或 RTABMAP               │    │
│  │ → nvblox (3D 避障 costmap)                     │    │
│  │ → coco_detector (物件偵測)                     │    │
│  │ → Qwen2.5-3B (輕量 LLM)                        │    │
│  │ → FunASR/Sherpa-onnx (STT)                     │    │
│  └─────────────────────────────────────────────────┘    │
└──────────────────────────┬──────────────────────────────┘
                           │ WiFi / 有線 (<200ms)
                           ↓
┌─────────────────────────────────────────────────────────┐
│                    Go2 Pro (MCU)                        │
│      L1 LiDAR (輔助) + 動作執行 + 防摔 + B 站           │
│      SDK 只傳指令，不做重運算                           │
└─────────────────────────────────────────────────────────┘
```

### 感測器角色分工

| 感測器 | 位置 | 用途 |
|--------|------|------|
| RealSense D435i | Jetson 上 | 主要深度感知 + VSLAM + 避障 |
| L1 LiDAR | Go2 內建 | 輔助避障（遠距）+ 大範圍掃描 |
| 前置相機 | Go2 內建 | 備用影像、遠端監控 |
